# laravel-ai-guardrails Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `padosoft/laravel-ai-guardrails`, a standalone, deterministic, offline-first Laravel package that closes the prompt-injection guardrails gap around `laravel/ai` agents/tools. It provides four composable controls: (A) a **Tool Firewall** that re-scopes model-chosen tool arguments to the authenticated user server-side and validates them against the tool's own JSON schema (untrusted-input posture); (B) an **Input Middleware + injection audit** that screens prompts for obvious injection patterns, refuses without calling the model when matched, and append-only-logs EVERY attempt (the audit trail is the value, not the regex); (C) an **Output Handler** that treats the model response as untrusted — escapes/sanitizes HTML & markdown, validates structured-output fields, and composes `padosoft/laravel-pii-redactor` for the PII slice; (D) a **HITL Bridge** adapter that routes a model's destructive tool calls (refund/delete/email) through `padosoft/laravel-flow`'s `approvalGate()` instead of letting the model pull the trigger.

**Architecture:** A single shared core per control (R44 tri-surface discipline: every capability reachable from PHP + Artisan + — where it makes sense — an MCP/HTTP-friendly service), thin adapters on top. The package **composes** `padosoft/laravel-flow` and `padosoft/laravel-pii-redactor` as **suggested/optional** dependencies with graceful degradation (null-object bindings chosen at boot via `class_exists`), and depends on `laravel/ai` only through its public contracts (`Laravel\Ai\Contracts\Tool`, `Laravel\Ai\Tools\Request`, `Laravel\Ai\Prompts\AgentPrompt`, `Laravel\Ai\Responses\AgentResponse`). Everything in controls A/B/C is deterministic and unit-testable without an LLM. Every config feature is a toggle (default-OFF where it changes behaviour); every flag is tested in BOTH states (R43); audit stores are append-only (R immutability). Skeleton mirrors `padosoft/laravel-evidence-risk-review` exactly.

**Tech Stack:** PHP `^8.3`; `illuminate/*` `^13.0`; PHPUnit `^12.5` + Orchestra Testbench `^11.0` (NOT Pest — mirror the skeleton); Laravel Pint `^1.18`; PHPStan `^2.0`. Suggested: `padosoft/laravel-flow` `^1.0` (HITL), `padosoft/laravel-pii-redactor` `^1.0` (PII), `laravel/ai` (the host's AI SDK; declared as a `require` peer at `*`/dev so contracts resolve, see Task 0). CI matrix PHP 8.3/8.4/8.5 × Laravel 13.* with `composer validate` → `pint --test` → `phpstan` → `phpunit`.

## Verified upstream API (researched from the local clones — cite these, do not invent)

**`laravel/ai`** (from `AskMyDocs/vendor/laravel/ai/`):
- `interface Laravel\Ai\Contracts\Tool` — `description(): \Stringable|string`; `handle(\Laravel\Ai\Tools\Request $request): \Stringable|string`; `schema(\Illuminate\Contracts\JsonSchema\JsonSchema $schema): array<string,\Illuminate\JsonSchema\Types\Type>`.
- `class Laravel\Ai\Tools\Request implements Arrayable, ArrayAccess` — ctor `__construct(protected array $arguments = [])`; `all(mixed $keys = null): array`; `toArray(): array`; array access over `$arguments`. (No public setter; new args = `new Request([...])`.)
- Agent middleware: generated by `php artisan make:agent-middleware`; stub signature is `public function handle(\Laravel\Ai\Prompts\AgentPrompt $prompt, \Closure $next)` returning `$next($prompt)->then(function (\Laravel\Ai\Responses\AgentResponse $response) { ... })`. Middleware is declared by an agent implementing `Laravel\Ai\Contracts\HasMiddleware::middleware(): array` and is executed via `Illuminate\Pipeline`: `->send($prompt)->through($middleware)->then(...)` (see `src/Providers/Concerns/GeneratesText.php:44`). Middleware entries may be class instances or closures `fn(AgentPrompt $prompt, Closure $next)`.
- `class Laravel\Ai\Prompts\AgentPrompt extends Prompt` — readonly `agent`, `attachments`, `timeout`; public readonly `string $prompt` (from parent); `contains(string $string): bool`; `prepend/append/revise(string $prompt): AgentPrompt`.
- `class Laravel\Ai\Responses\AgentResponse extends Laravel\Ai\Responses\TextResponse` — `public string $text`; `public Usage $usage`; `public Meta $meta`; `Collection $toolCalls`, `Collection $toolResults`, `Collection $steps`, `Collection $messages`; `then(callable): self`; `__toString()` returns `$text`. **Note: `$text` is a public mutable property** — control C rewrites it in place.
- `class Laravel\Ai\Responses\Data\Usage` (`promptTokens`, `completionTokens`, …); `class …\Data\Meta` (`?string $provider`, `?string $model`, `Collection $citations`); `class …\Data\ToolCall` (`string $id`, `string $name`, `array $arguments`); `class …\Data\ToolResult` (`string $id`, `string $name`, `array $arguments`, `mixed $result`).
- Events: `Laravel\Ai\Events\InvokingTool` (`invocationId`, `toolInvocationId`, `Agent $agent`, `Tool $tool`, `array $arguments`) and `ToolInvoked` (adds `mixed $result`).
- `class Laravel\Ai\Schema` wraps `Illuminate\JsonSchema\Types\Type`; tool schemas return `array<string,Type>` keyed by argument name. `Type::toArray()` yields a JSON-schema fragment (`['type'=>'string', 'required'=>bool, ...]`) usable for deterministic validation.

**`padosoft/laravel-flow`** (composer name `padosoft/laravel-flow`; provider `Padosoft\LaravelFlow\LaravelFlowServiceProvider`; facade `Padosoft\LaravelFlow\Facades\Flow`):
- `Padosoft\LaravelFlow\FlowDefinitionBuilder::approvalGate(string $name): self` (calls `step($name, ApprovalGate::class)->withDryRun(true)`).
- `Flow::define(string $name): FlowDefinitionBuilder` → `->withInput(array)->step(string,$handlerFqcn)->approvalGate(string)->step(...)->register()`.
- `Flow::execute(string $name, array $input): FlowRun` — pauses at the gate; `FlowRun::STATUS_PAUSED`; the issued plain token is `$run->approvalTokens[$stepName]->plainTextToken` (DTO `Padosoft\LaravelFlow\IssuedApprovalToken`: `plainTextToken`, `tokenHash`, `expiresAt`, `runId`, `stepName`, `approvalId`).
- `Flow::resume(string $token, array $payload = [], array $actor = []): FlowRun` and `Flow::reject(string $token, array $payload = [], array $actor = []): FlowRun`.
- Handlers implement `Padosoft\LaravelFlow\FlowStepHandler::execute(Padosoft\LaravelFlow\FlowContext $context): Padosoft\LaravelFlow\FlowStepResult`; `FlowContext::$input` is the array; results via `FlowStepResult::success(array)`, `::failed(\Throwable)`, `::paused(array)`.
- Append-only audit model `Padosoft\LaravelFlow\Models\FlowAuditRecord` (table `flow_audit`, throws on update/delete); approvals table `flow_approvals` / model `FlowApprovalRecord`. Authorizer contract `Padosoft\LaravelFlow\Dashboard\Authorization\DashboardActionAuthorizer` (`canApproveByToken`, `canRejectByToken`). Persistence + a non-array cache lock store are REQUIRED for approvals.

**`padosoft/laravel-pii-redactor`** (composer name `padosoft/laravel-pii-redactor`; provider `Padosoft\PiiRedactor\PiiRedactorServiceProvider`; facade `Padosoft\PiiRedactor\Facades\Pii`; alias `Pii`; container key `pii-redactor`; engine `Padosoft\PiiRedactor\RedactorEngine`):
- `Pii::redact(string $text, ?Padosoft\PiiRedactor\Strategies\RedactionStrategy $override = null): string` — deterministic, offline.
- `Pii::scan(string $text): Padosoft\PiiRedactor\Reports\DetectionReport` — `total(): int`, `isEmpty(): bool`, `countsByDetector(): array`, `toArray(): array`.
- `Pii::isEnabled(): bool`. Config master toggle `pii-redactor.enabled`.

```php
<?php

declare(strict_types=1);

namespace Padosoft\AiGuardrails\Firewall;

use Illuminate\JsonSchema\JsonSchema;
use Laravel\Ai\Contracts\Tool;
use Padosoft\AiGuardrails\Contracts\ToolArgumentValidator;

final readonly class SchemaToolArgumentValidator implements ToolArgumentValidator
{
    public function __construct(private bool $rejectUnknown = true) {}

    public function validate(Tool $tool, array $arguments): array
    {
        $schema = $tool->schema(new JsonSchema);
        $errors = [];

        foreach ($schema as $key => $type) {
            $definition = $type->toArray();
            $required = ($definition['required'] ?? false) === true;
            $exists = array_key_exists($key, $arguments);

            if (! $exists) {
                if ($required) {
                    $errors[$key] = "Missing required argument [{$key}].";
                }

                continue;
            }

            $expected = $definition['type'] ?? null;
            if (is_string($expected) && ! $this->matchesType($expected, $arguments[$key])) {
                $errors[$key] = "Argument [{$key}] must be of type [{$expected}].";
            }
        }

        if ($this->rejectUnknown) {
            foreach (array_keys($arguments) as $key) {
                if (! array_key_exists($key, $schema)) {
                    $errors[$key] = "Unknown argument [{$key}] is not declared in the tool schema.";
                }
            }
        }

        return $errors;
    }

    private function matchesType(string $expected, mixed $value): bool
    {
        return match ($expected) {
            'string' => is_string($value),
            'integer' => is_int($value),
            'number' => is_int($value) || is_float($value),
            'boolean' => is_bool($value),
            'array' => is_array($value),
            'object' => is_array($value),
            default => true, // unknown schema type → do not block (fail-open on type only)
        };
    }
}
```

<?php

declare(strict_types=1);

namespace Padosoft\AiGuardrails\Firewall;

use Closure;
use Illuminate\Contracts\JsonSchema\JsonSchema;
use Laravel\Ai\Contracts\Tool;
use Laravel\Ai\Tools\Request;
use Padosoft\AiGuardrails\Contracts\ArgumentScoper;
use Padosoft\AiGuardrails\Contracts\ToolArgumentValidator;
use Padosoft\AiGuardrails\Exceptions\ToolArgumentRejection;
use Stringable;

final class FirewalledTool implements Tool
{
    /** @param Closure():(int|string|null) $principalResolver */
    public function __construct(
        private readonly Tool $delegate,
        private readonly ArgumentScoper $scoper,
        private readonly ToolArgumentValidator $validator,
        private readonly Closure $principalResolver,
    ) {}

    public function description(): Stringable|string
    {
        return $this->delegate->description();
    }

    public function handle(Request $request): Stringable|string
    {
        // 1. Re-scope owner-controlled keys to the authenticated principal (server-side truth).
        $scoped = $this->scoper->scope($request->toArray(), ($this->principalResolver)());

        // 2. Validate the (now scoped) arguments against the tool's own schema — untrusted input.
        $violations = $this->validator->validate($this->delegate, $scoped);
        if ($violations !== []) {
            throw new ToolArgumentRejection($violations, (string) $this->delegate->description());
        }

        // 3. Delegate with a fresh, sanitized Request.
        return $this->delegate->handle(new Request($scoped));
    }

    /** @return array<string, \Illuminate\JsonSchema\Types\Type> */
    public function schema(JsonSchema $schema): array
    {
        return $this->delegate->schema($schema);
    }
}
